Introduction

Skin, Ear, Nose, and Throat diseases and disorders are prevalent in the world, affecting a vast majority of the world population. We perceived and pondered this problem how we could solve this problem and came up with the idea of using machine learning to examine various pictures of skin diseases and disorders, and diagnose users with a particular skin condition predicted from the machine learning model. This is SENT.it. Since it is difficult to take a picture of the ear, nose, and throat, we used a different type of machine learning algorithm to help diagnose users with a particular ear, nose, or throat condition using their symptoms. Users select their symptoms and the machine learning algorithm predicts the ear, nose, or throat condition based off those symptoms. Aside from predicting skin, ear, nose, and throat conditions using machine learning algorithms, we also wanted to create a communication service for users to communicate with their doctors about their results. This helps for patients to quicken their diagnosis and start on treatment if available.

Beginnings

Skin Data

To collect the skin data to train our skin machine learning model, we used a dataset platform called Kaggle. Kaggle is a website used by people involved in data science and machine learning to share data that can be used for data science and machine learning projects. We used Kaggle to find various image datasets for different skin conditions to train our skin machine learning model to diagnose user-sent images of skin diseases or disorders.

Ear, Nose, and Throat Data

Since it is difficult to take pictures from inside a one's ear, nose, or throat, we needed a different strategy to help diagnose ear, nose, and throat conditons. Instead of using images to train the machine learning model, we used lists of symptoms associated with a particular disease or disorder. To do this, we created our own datasets for the machine learning model to train on by web scraping websites such as Malacards and Diseases Database.

Name Origin

So, where did the name "SENT.it" come from? Well, our project is based around skin, ear, nose, and throat diseases, so we decided to put the first letters of those four words together: Skin + Ear + Nose + Throat = SENT. This acronym applies to our project because we also have a communication portion of the project. Finally, we want users to send the data that they get from the machine learning algorithms to their doctors, so thus, SENT.it was born!

Tools

Data Science

Before we began to investigate the different machine learning algorithms to use, we needed to organize the data and visualize it to help us incorporate the machine learning algorithms into the data. We mainly used four tools: pandas, NumPy, matplotlib, and BeautifulSoup.

We used pandas, a python data science library, to organize data into tables to make it more easily readable, as well as to remove any missing data that could affect how the machine learning algorithm could interpret the data.

We also used NumPy, another python data science library, to transfer data into n-dimensional arrays (ndarrays), a commonly used data type in NumPy. ndarrays are integral in performing mathematical calculations over large amounts of data. Since we have a large about of data to manipulate and train the machine learning model, NumPy is extremely important in preparing the data for machine learning.

Matplotlib, a python data science library for visualizing data, was also important in how we interpreted the data before putting it into the machine learning algorithm. Matplotlib can make many types of graphs such as histograms, bar graphs, and scatterplots, and allows for flexibility in data to make visualizing it so much easier.

Finally, we used BeautifulSoup, a python data science library, to help with pulling out data from HTML and XML files. This was helpful in creating out symptoms model because it required a lot of data to be pulled out of websites, also known as webscraping, and BeautifulSoup was integral in doing this. This quickened the process of extracting data from websites because we could not use already-made datasets for this portion since there were none made already.

Machine Learning

Now that our data was put together and organized in the most optimized way, it was time to implement machine learning algorithms. For SENT.it, we used two main machine learning algorithms: vision transformer for the skin model and normal neural networks for the ear, nose, and throat models.

The foundation of SENT.it is its implementation of machine learning algorithms, in specific, Vision Transformer or ViT. ViT is an algorithm that is an alternative to Convolutional Neural Networks, also known as CNNs. Its heavy usage in image recognition has earned its reputation as state-of-the art model that outperforms CNN's nearly four times.

A Vision Transformer (ViT) is a deep learning model that uses the significance of each data input to do the mechanism of image recognition. The basic process that makes it so useful compared to other algorithms such as CNN, is something out of the ordinary-- it uses CNN as its foundation! When a transformer is combined with a CNN front end, the CNN turns standard pixels into a feature map, when is then turned into a sequence of tokens by a tokenizer, a way to convert sentences into smaller units (such as words). This is then put into the transformer (this is where ViT is implemented!). The transformer then uses the technique of attention, which quantifies entities into a hierarchy in order to classify data. After this technique is used, these tokens are transformed into output tokens that reconnect to the feature map. This is crucial for getting all of the small details that CNN’s may not be able to obtain.

Compared to CNN, this allows for not as many tokens to be used and thus lowers the price of image recognition, making it a great alternative to CNN!

To implement ViT we used Hugging Face, a company specialized in making tools for the implementation of machine learning. We used the Transformers library of Hugging Face to do this which allowed for ViT to be used.

Now, for our ear, nose, and throat models, we used normal neural networks. This is a standard machine learning algorithm and since neural networks can get complex pretty quickly, we plan to use a new algorithm for these models. One of our ideas so far is an algorithm called XGBoost which helps to lower the time it takes for the model to complete a task.

Challenges

Finding the right data

There is so much data that we could use for our project, and it was difficult to pick which type of data we wanted to use for each part of the project. To help find the right data, we weighed out all of the datasets we could use and picked datasets based on the type of machine learning algorithm we used.

Picking the right Machine Learning algorithm

This part was very difficult. We went from algorithm to algorithm trying to find which one would fit the data the best and have the best results. We started of with a CNN, which is a common algorithm used for image recognition, but eventually transitioned to ViT for its power and speed. We had to go through many steps to find possible machine learning algorithms for our problem, and we are still trying to optimize the time it takes for the algorithm to complete a task accurately.

Front-end Problems

We had many challenges with the front-end portion of our project because we wanted to create something minimalistic and simple to use, but also include functionality and interactivity. We required many packages that were not included in the base HTML/CSS/JS code such as Jinja, a templating language used for python implementation in HTML code. Flask, a python micro-framework, was also used to implement python code into HTML. To wireframe the interactivity of the website, we used Figma to plan out the structure and interactivity of the website and we implemented it using Webflow.

Communication
Signup and Login
‍

One of the biggest features of SENT.it is the communication portion. We required some way of communication between doctors and patients and we needed to make this as simple as possible. When users are signing up for SENT.it as either a medical practitioner or patient, they need to have different interfaces based on the type of user. For instance, doctors need to have their interface as being able to communicate with multiple patients if their patients are using SENT.it, and users must have an interface that gives them the ability to use the machine learning algorithm in order to identify the image sent for the skin model, or the list of symptoms they give for the ear, nose, and throat models.

Furthermore, one of the features SENT.it uses is its location services. By submitting both a location for both medical practitioner and patient users, we are able to connect doctors and patients that are nearby. This allows for users to easily communicate with doctors that are nearby, and get in-person communication and/or treatment if needed.

Submitting

After users sign up for their account, be it a patient or medical practitioner. If the user is a patient, the user will be able to use a form to either submit a picture of the skin area they are concerned about, or submit a list of symptoms that they have for their ears, nose, or throat, and the machine learning algorithms will predict the condition they have and give an accuracy of the prediction. The user will then have the option to learn more about the condition predicted by the machine learning algorithm and send the results to their doctor for further confirmation of the condition or possible treatments.

Messaging

After users submit their images or list of symptoms to the machine learning algorithms, they can communicate with their doctors to plan out treatments or confirm the diagnosis. SENT.it has a messaging platform for communication between doctors and patients that allows for easy transfer of data. Also, with the location feature, doctors are able to figure out how to schedule treatments with their patients.

The messaging interface for patients looks like a standard messaging platform. Patients have an option to send an already-made prompt with their diagnosis and data already predicted by the machine learning algorithms, and doctors can send messages back. This makes it easier for both patients and doctors because both the communication and function of our project is contained in the same website.


While the messaging UI is functional at this point, we are still looking to make improvements to it such as adding the functionality of video calls and more types of data transfer.
